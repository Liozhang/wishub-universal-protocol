# WisHub Glossary - English (Source)

This glossary defines key terminology used in WisHub protocol documents.

---

## Core Concepts

| Term | Definition |
|------|-----------|
| **WisHub** | Wisdom Connection Hub - An open, scalable, and high-performance knowledge sharing and intelligent service ecosystem |
| **WisUnit** | Wisdom Unit - Standardized knowledge unit with three-layer structure (executable, structured, natural language) |
| **WISE** | WisStore, WisSync, WisVerify, WisIncentive, WisDedup, WisCache - Multi-level storage system |
| **Hub** | Connection Center - Centralized storage, intelligent distribution, global indexing |

## WisUnit Terms

| Term | Definition |
|------|-----------|
| **Executable Layer** | Layer 1 of WisUnit - AI/machine/Agent-executable knowledge (code, algorithms, functions) |
| **Structured Layer** | Layer 2 of WisUnit - Program/system-processable knowledge (JSON, schemas, metadata) |
| **Natural Language Layer** | Layer 3 of WisUnit - Human-understandable knowledge (descriptions, explanations, narratives) |

---

## WISE Terms

| Term | Definition |
|------|-----------|
| **L1 Storage** | Level 1 - In-memory storage (fast access, temporary) |
| **L2 Storage** | Level 2 - Distributed storage (cache, synchronization) |
| **L3 Storage** | Level 3 - Persistent storage (databases, backups) |
| **WisStore** | Storage component of WISE system |
| **WisSync** | Synchronization component of WISE system |
| **WisVerify** | Validation component of WISE system |
| **WisIncentive** | Incentive component of WISE system |
| **WisDedup** | Deduplication component of WISE system |
| **WisCache** | Caching component of WISE system |

---

## Agent Terms

| Term | Definition |
|------|-----------|
| **Agent** | Intelligent entity that can execute tasks, interact with other agents, and use WisUnits |
| **Task Agent** | Agent type focused on task execution |
| **Orchestration Agent** | Agent type focused on coordinating other agents |
| **Agent Registration** | Process of registering an agent with the Hub |
| **Agent Invocation** | Process of calling an agent to execute a task |
| **Quality Incentive** | Reward mechanism for high-quality agent performance |

---

## MCP Terms

| Term | Definition |
|------|-----------|
| **MCP** | Model Context Protocol - Protocol for connecting AI models with knowledge bases |
| **MCP Session** | A session for querying knowledge and injecting context into AI models |
| **Context Injection** | Process of injecting knowledge into AI model context |
| **Knowledge Query** | Query for retrieving relevant knowledge from the Hub |

---

## Skill Terms

| Term | Definition |
|------|-----------|
| **Skill** | Reusable capability that can be called by Agents |
| **Skill Registration** | Process of registering a skill with the Hub |
| **Skill Discovery** | Process of finding available skills |
| **Skill Orchestration** | Process of coordinating multiple skills in a workflow |

## Security Terms

| Term | Definition |
|------|-----------|
| **API Key** | Authentication key for accessing WisHub APIs |
| **JWT** | JSON Web Token - Token-based authentication |
| **Zero-Knowledge Proof** | Cryptographic proof that reveals no information beyond the statement being proven |
| **Encryption** | Process of encoding information to prevent unauthorized access |

## Economy Terms

| Term | Definition |
|------|-----------|
| **WIS Token** | Native token of WisHub ecosystem |

| **Governance** | Token holder voting on protocol upgrades and policy changes |
